§ 2 — Objective

Project Objective

This project demonstrates how to build a fully automated, end-to-end restaurant intelligence workflow capable of replacing hours of manual research with a reproducible pipeline. The goal is to surface the best dining venues in the Ari Zone for a corporate team of 8–12 people — balancing quality, budget, logistics, and group dynamics — with zero manual scraping or subjective guesswork.

🤖

Automate Discovery

Use Apify scrapers to collect live restaurant data from Google Maps without manual effort.

⚙️

Orchestrate via n8n

Webhook-triggered automation routes and transforms raw data into a structured spreadsheet.

🧠

AI-Powered Scoring

Claude AI applies a 6-dimension, 100-point scoring rubric to rank every venue objectively.

📊

Decision-Ready Output

Deliver ranked tables, trade-off analysis, and top-3 recommendations ready for team decision-making.

§ 3 — Automated Workflow

Pipeline Overview

The six-stage pipeline flows from raw scraping through orchestration and AI analysis to final HTML delivery. Each stage has a distinct responsibility and hands off clean, structured data to the next.

🕷️ Apify Scraper Crawls Google Maps for restaurants in Ari Zone. Returns structured JSON.
→
🔔 Webhook Apify posts completed dataset payload to an n8n webhook endpoint.
→
⚙️ n8n Automation Parses, deduplicates, and normalises fields. Routes data to Google Sheets.
→
📋 Google Sheets Acts as the live data warehouse. Stores raw + cleaned records per run.
→
🧠 Claude AI Analytics Applies 100-point scoring model. Generates rankings and trade-off analysis.
→
🌐 HTML Report Final output hosted on GitHub Pages. This document.

Trigger Mechanism

The pipeline is event-driven: Apify's "Run Finished" webhook fires a POST request to the n8n endpoint, passing the dataset ID. n8n retrieves the dataset via the Apify API, processes it, and appends rows to Google Sheets in real time — no manual intervention required after the initial setup.

§ 6 — Scoring Model

Scoring Rubric — 6 Weighted Dimensions (100 pts)

Each restaurant is scored independently across six dimensions. Weight distribution is fixed per project specification and must not be altered.

⭐ Review & Rating Consistency 25 pts

Authenticity and alignment between rating score and implied review volume. A 5.0 rating with suspected low review count is penalised for lower credibility. Scores range from 11 (4.3★) to 23 (4.9★); 5.0★ scores 18 due to inflated-rating risk.

👥 Group Suitability 20 pts

Physical capacity and format appropriateness for 8–12 people. BBQ/hot-pot formats score highest (17–19). Intimate bistros, salad bars, or rooftop kiosks score lower due to seating constraints or awkward group ordering.

💰 Price & Value 15 pts

Cost-effectiveness relative to experience quality. ≤฿300/person = 13–15 pts; ฿300–600 = 9–12; ฿600–1,000 = 6–8; >฿1,000 = 3–5. Inferred prices (missing from dataset) receive a 1–2 pt penalty for uncertainty.

🚇 Travel & Accessibility 15 pts

Proximity to BTS Ari station and ease of navigation. Main road (Phahonyothin Rd) = 13–14; accessible soi within 5 min walk = 11–12; deep or complex soi = 9–10; far from BTS (Rama VI) = 8–9.

📋 Data Quality & Completeness 15 pts

Richness of the dataset record. Scraper entries (full URL + price + address) = 13–14 pts. Places API entries (place ID only, no price range) = 7 pts. Penalises data gaps that require inference.

✨ Uniqueness 10 pts

Presence of a signature dish, theatrical experience, or distinct selling point. Live-fire teppanyaki, rooftop views, or niche cuisines score 8–10. Generic Thai or standard burgers score 4–6.

Noise Level: Sizzling, exhaust fans, and lively crowd make structured conversation (planning, retrospectives) essentially impossible.
Reservation Required: Walk-in for 8–12 is nearly impossible. Book 3–5 days ahead; risk of being split across 3 non-adjacent tables.
Dietary Exclusion: Heavily meat-centric. Vegetarians, vegans, and non-beef/pork eaters have very limited options — highest dietary-exclusion risk of the three.

✅ What Went Well

System Architecture & Hybrid Workflow

The project successfully implemented a hybrid architecture by separating data collection (Apify), orchestration (n8n), storage (Google Sheets), and intelligence/scoring (Claude Cowork). This modular approach made the system stable, saved API credits, and allowed each stage to be tested independently.

Effective Data Collection & Automation

The automated pipeline functioned successfully end-to-end. Apify reliably scraped 29–30 restaurant records focused on the Ari Zone with consistent data schemas, while operating within the free credit limits.

AI Analysis & HTML Output Completeness

Claude accurately executed the 6-dimension scoring rubric, producing non-trivial rankings and actionable trade-offs rather than surface-level praise. The final HTML output was complete and incorporated all 9 mandatory sections.

⚠️ Automation Bugs & Data Issues

Data Structure Conflicts & Missing Fields

Raw data from different sources had mismatched JSON structures (e.g., title vs. name), causing data mapping issues in n8n. Crucial fields like Source URLs and Price Ranges were also missing, disrupting scoring calculations.

Google API Authentication Hurdles

Multiple Google Sheets API issues arose: OAuth2 token expiration, security blocks from the app being stuck in "Testing Environment," and errors caused by failing to enable the API in Google Cloud Console.

Integration & Webhook Limitations

Technical hurdles included 404 errors in n8n from dynamic Dataset ID mismatches, webhook timeouts exceeding n8n's 30-second limit from large Apify payloads, and the financial risk of draining Apify's limited $5 free credit.

💡 Lessons Learned & Mitigation Strategies

Resolving Data Conflicts via AI & Restructuring

Data conflicts were resolved by separating n8n's "Append row in sheet" nodes per source. Claude AI was then used to clean data, fill in missing Google Maps URLs, and normalise price ranges. The 404 error was bypassed by temporarily hardcoding the Dataset ID in n8n.

Overcoming API & Authentication Bottlenecks

OAuth2 security hurdles were bypassed by switching to a Service Account (JSON Key) and granting "Editor" access directly to the bot's email. Key lesson: always ensure APIs are fully enabled in Google Cloud Console before running data pipelines.

Optimizing Resources & Webhook Stability

Webhook timeouts were mitigated by switching to Apify dataset polling. Budget bleed was prevented by capping scraper output at 25–30 results and disabling individual review fetching. Future recommendation: extract price data at the scraping stage and add n8n data validation before writing to Sheets.
